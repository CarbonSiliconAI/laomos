# LLM Chat (Fallback)

A universal fallback skill that handles any task by directly conversing with the cloud LLM.

## When to Use

This skill is automatically used when:
- No specialized skill matches the user's request
- The task requires general knowledge or reasoning  
- A creative, analytical, or conversational response is needed

## Capabilities

- Answer questions on any topic
- Summarize text or documents
- Generate creative content (emails, messages, stories)
- Analyze data or code
- Translate between languages
- Provide step-by-step instructions
- General reasoning and problem-solving

## Usage

Simply describe your task or question in natural language. The LLM will process it and return a response.

**Example:**
User: Summarize the key points of this article about AI safety
User: Write a professional email declining a meeting invitation
User: Explain the difference between TCP and UDP

## Context Integration - CRITICAL REQUIREMENT

When generating responses that depend on external data (such as weather information, location data, or other action results), you MUST:

1. **Explicitly Reference Retrieved Data**: Include specific values (temperatures, conditions, forecasts, humidity, wind speed) directly in your recommendations
2. **Create Weather-to-Recommendation Mappings**: 
   - For temperatures below 50°F: recommend warm layers, jackets, long pants
   - For temperatures 50-65°F: recommend light jackets, sweaters, transitional pieces
   - For temperatures above 65°F: recommend lighter clothing, short sleeves
   - For sunny conditions: mention sun protection, lighter colors
   - For high humidity: avoid heavy fabrics, recommend breathable materials
   - For wind: recommend windproof outer layers
3. **Validate Data Availability**: Before generating output, explicitly confirm that all required contextual data from previous actions is present and accessible
4. **Integrate All Parameters**: Weather-specific recommendations must incorporate temperature, humidity, wind, and condition data - not generic templates
5. **Avoid Generic Templates**: Do not reuse identical "body type" or "color palette" sections across different weather contexts
6. **Document Data Usage**: Explicitly state which weather parameters influenced each recommendation

### Example of Correct Integration
**WRONG**: "Here are clothing recommendations for any season..."
**CORRECT**: "Based on the current 44°F sunny weather with 82% humidity and 5mph wind in Union City, California, I recommend: [specific items tailored to these exact conditions]"

## Error Handling

If the LLM Chat skill is invoked for tasks that require:
- Real-time data fetching (weather, stock prices, news)
- System command execution
- External API calls with authentication
- File system operations

The skill will decline and recommend using a specialized skill instead.

## Notes

- No external tools, APIs, or commands are needed
- Responses are generated by the cloud LLM model
- Best for tasks that don't require local system access or specific tool execution
- **MANDATORY**: Complete context data from prerequisite actions must be explicitly provided and referenced in dependent outputs
- **MANDATORY**: Data integration is not automatic - prompts must explicitly require weather-based customization
- Not suitable as a replacement for specialized skills that handle specific tool integrations
- When chaining actions, ensure the prompt for dependent actions explicitly includes retrieved data and requires its use in the output
